## 2026-03-25 - Custom functions over regex for exact or simple class matching
**Learning:** `BeautifulSoup` searches using `class_=re.compile(...)` execute significantly slower than native string matching or custom evaluation functions because BeautifulSoup evaluates the regex machinery inside its internal attribute-parsing loops. Passing an exact string `class_="foo"` is ~3x faster. For OR-conditions (`foo|bar`), a custom function evaluating the class array directly (`any(c in (...) for c in classes)`) is ~3x faster than compiling and executing `(?i)foo|bar`.
**Action:** When filtering elements in BeautifulSoup by class names using exact match or a limited set of known substrings, always prefer native strings (`class_="exact"`) or a fast custom evaluation function passed directly to `soup.find()`. Avoid `class_=re.compile(...)` to bypass the slow regex and attribute resolution overhead entirely.
